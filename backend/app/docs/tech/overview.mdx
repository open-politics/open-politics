## SSARE Data Engine

The Semantic Search Article Recommendation Engine (SSARE) is the backbone of our data processing and storage system. Key features include:

- **Scraping and Processing**: Flexible integration of various news sources and scraping scripts.
- **Vector Representation**: Advanced embedding techniques for semantic understanding of articles.
- **Named Entity Recognition**: Extraction of key entities like locations, organizations, and people.
- **Geocoding**: Mapping of recognized locations for geospatial analysis.
- **Storage**: Efficient storage in both PostgreSQL and Qdrant vector database for optimized retrieval.

## Web Application

Our web application serves as the user-facing interface for Open Politics. It's built with:

- **Frontend**: Next.js for a responsive and dynamic user interface.
- **Backend**: FastAPI for efficient API handling and data serving.
- **Visualization**: Integration of tools like amcharts for our "Open Globe" interface.
- **UI Components**: Utilization of shadcn UI library for consistent and accessible design.

## AI Models

We employ a variety of AI models and techniques to power our political analysis:

- **Language Models**: Leveraging open-source models for text understanding and generation.
- **Embedding Models**: For semantic search and article recommendation.
- **Custom AI Chains**: Engineered for political analysis tasks and tool calling.
- **Knowledge Graphs**: Automatically generated and updated for complex relationship mapping.

Our technology stack is designed to be modular, scalable, and open for community contributions, ensuring that Open Politics can evolve to meet the changing needs of political information analysis.
